# Project Identity

## Product

- Odoo 19 Implementation Control Platform
- guided implementation platform that reaches usable Odoo implementation outcomes through governed application-layer execution
- takes a business from scoped setup through truthful, usable implementation
- supports Community and Enterprise
- supports Odoo Online, Odoo.sh for Enterprise only, and On-Premise within the stated scope rules

## End Goal

- usable guided implementation with real Odoo writes through governed application-layer execution
- every in-scope wizard/domain must truthfully write or be explicitly marked manual/out-of-scope with exact reason
- frontend/shell/dashboard work is subordinate to implementation write capability
- preview/approval/execution recording are means to real implementation, not the end product

## Core Promise

- guide a project through correct Odoo 19 setup in the right order
- require confirmation at critical checkpoints
- explain downstream impact before decisions are locked in
- safely apply real Odoo application-layer writes through governed bounded execution
- record truthful execution results with audit traceability
- reach a usable implementation state with enough domain coverage and data setup
- support only forward-safe implementation activity

## Target Users

Primary users:

- implementation leads
- functional consultants
- solution architects
- project owners
- client-side process owners
- administrators responsible for controlled configuration

Secondary users:

- trainers consuming optional guidance outputs
- reviewers approving checkpoint completion
- technical teams implementing bounded configuration tasks under governed rules

## Product Principles

- control before configuration
- validation before progression
- guidance with consequences
- preview before execution
- approval before execution
- bounded application-layer execution only
- forward-safe expansion only
- explicit boundaries
- human decision ownership
- operational readiness is separate from configuration completion

## What The Product Is

- a guided implementation platform that reaches usable implementation outcomes through governed execution
- an answer-driven discovery and configuration determination system
- a checkpoint and validation framework
- a preview-before-execution and approval-before-execution enforcement layer
- a bounded application-layer execution surface for real Odoo writes
- an audit-traceable execution recording system
- a mechanism to support fresh implementations and forward-safe expansion only

## What The Product Is Not

- a remediation tool
- a data repair tool
- a migration-fix platform
- a developer diagnostic workbench
- a transactional correction engine
- an unrestricted configuration automation tool
- a raw database writer
- a shell-first dashboard project
- a guide-only planner that stops before real Odoo writes
- a substitute for business ownership of key implementation decisions

## Success Definition

- start an Odoo 19 implementation or forward-safe expansion with a clearly identified project mode
- move through implementation stages in a governed order
- complete critical checkpoints with appropriate validation evidence
- understand downstream impact before making consequential configuration decisions
- execute real Odoo application-layer writes through governed bounded execution
- record truthful execution results
- reach a usable implementation state with enough domain coverage and data setup
- separate required, go-live, recommended, and optional work clearly
- resume in-progress work with accurate project state, decision history, and execution history
- maintain clean scope boundaries without drifting into remediation or diagnostic behavior

A wizard or domain is not "done" until it can truthfully write through governed execution, or is explicitly marked manual/out-of-scope with exact reason.
