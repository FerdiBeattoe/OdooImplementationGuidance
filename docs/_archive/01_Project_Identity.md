# Project Identity

## What This Product Is

A human-facing Odoo 19 self-implementation platform that helps people implement Odoo without consultants.

The target user is a business owner, operations lead, or in-house team who wants to implement Odoo themselves. The platform replaces the need for an external consultant by guiding the user through the full implementation in plain language.

## The Three Fixed Product Surfaces

These surfaces are the product. All three must be present and functional.

### Pipeline

The logic and control layer. It sequences implementation stages, drives discovery questions, determines domain activation, enforces checkpoints, governs execution eligibility, and tracks implementation state. Users see their progress, next actions, and blockers here.

### Module Dashboard

The module-level workspace. It exposes each domain's checkpoints, guidance, inspection output, preview, and execution surface. Where truthful governed writes exist, the Module Dashboard writes configuration directly to Odoo through approved, audited execution.

### Import Wizard

The data import surface. It guides the user through data preparation and writes records to Odoo through governed import execution.

## Core Promise

The platform asks the right questions, explains every decision in plain language, writes what it can truthfully write to Odoo, and guides the user through the remaining setup and import work — until the implementation is complete and operationally ready.

## What This Product Is Not

- A remediation, repair, or migration-fix tool
- A diagnostic or developer workbench
- A connector or integration middleware platform
- A generic Odoo admin console
- A guide-only planner that stops before real Odoo writes
- A consultant engagement platform
- An AI assistant or general-purpose knowledge tool

## Odoo Scope

- Version: Odoo 19 only
- Editions: Community and Enterprise only
- Deployments: Odoo Online, Odoo.sh Enterprise, On-Premise only
